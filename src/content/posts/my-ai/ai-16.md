# AI 辅助的前端开发工作流系统开发, 要支持自动化完成代码审查、测试用例生成、文档编写等重复性工作。

考察点

- 前端传统型工程化-现代化 AI工程化
- AI 代码审查
- 测试用例自动生成
- 文档方案自动生成维护

## 思路

AI 辅助开发工作流的设计需要覆盖前端开发的全生命周期。从代码编写阶段的智能提示和自动补全、vibecoding，到代码审查阶段的质量检查和优化建议，再到测试阶段的用例生成和执行，最后到文档阶段的自动编写和维护，每个环节都可以通过 AI技术来提升效率。

代码编写阶段，codex、claude。同时要实现代码规范检查，通过AI分析来确保代码符合项目的编码规范。可以使用 ESLint、Stylelint 等工具进行基础检查,然后通过 AI 进行更高级的代码质量分析。

代码审查是保证代码质量的重要环节。传统的人工审查耗时耗力，AI可以帮助自动化这个过程。AI代码审查应该包括:功能正确性检查(验证代码是否实现了预期功能)、性能优化建议(识别潜在的性能问题)、安全漏洞检测(发现可能的安全风险)、代码风格检査(确保代码风格的一致性)。

测试用例的自动生成是提高测试效率的关键。可以基于代码逻辑和需求文档来生成测试用例，包括单元测试、集成测试和端到端测试。对于前端应用，特别要关注UI交互测试，可以使用 Playwright、Cypress等工具结合 AI 来生成和执行测试脚本。

- unit test, -> vitest
- e2e test, -> Playwright

文档自动化是减少文档编写工作量的有效方法。可以基于代码注释、接口定义、需求文档等信息自动生成技术文档。同时要实现文档版本管理，当代码发生变更时，自动更新相关的文档内容。文档应该包括 API 文档、技术架构说明、开发指南、用户手册等。

工作流的集成需要与现有的开发工具和流程无缝对接。应该支持主流的代码编辑器(VS Code、Webstorm 等)、版本控制系统(Git)、构建工具(Webpack、Vite等)、CI/CD平台(Jenkins、GitLab CI、阿里云效 等)。通过插件或 API 的方式来集成这些工具，实现自动化的工作流程。流水线定义AI的卡点
