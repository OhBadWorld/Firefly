# 对多 Agent 系统的，结合前端业务场景，告诉你多 Agent 的核心设计思路、优势，以及常用的落地框架。

考察点:

- AI 应用开发知识广度
- 复杂业务 AI 方案设计能力
- 多 Agent 协作开发经验

多 Agent，为了隔离上下文，分工明确

## 设计思路

- 明确分工:给每个 Agent 设定专属的人设、职责、能力边界，比如有的 Agent 负责需求规划，有的负责资料检索，有的负责代码编写，有的负责测试验
- 收标准化通信:基于 langgraph 看看 subAgent，制定统一的信息传递规则，让 Agent 之间可以顺畅传递信息、对齐目标、同步进度、反馈问题
- 流程管控:主 agent，负责任务分配、进度管控、结果汇总，保证整个系统按目标推进

### 核心优势

- 复杂任务更稳定:把复杂任务拆成多个专项子任务，每个Agent 只负责自己擅长的部分，避免单个大模型能力过载出现幻觉
- 逻辑更清晰、可维护性更强:每个 Agent 的职责、提示词、能力都可以独立迭代优化，不用修改整个系统
- 可拓展性强:新增能力只需要新增对应的专项 Agent，无需改动原有系统

微内核设计/插件化设计思想【mcp、skill、subAgent】

### 落地框架

LangChain、LangGraph、AutoGen、CrewAI

### 前端落地

比如电商平台的智能客服系统，用多 Agent 实现:需求理解 Agent 负责接待用户、提取核心问题，售后 Agent 负责处理退换货，订单 Agent 负责查物流 /订单，商品 Agent 负责推荐商品，最终汇总结果给用户，比单个 Agent的处理准确率和稳定性高很多。
