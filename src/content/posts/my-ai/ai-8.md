# Agent 的核心底层逻辑 Rect概念, 简单说明大模型的基础架构 Transformer，以及作为前端开发，我们在 AI 落地中需要关注的核心点

考察点

- Agent 底层运行逻辑本质理解
- 大模型基础架构认知
- 不要细节算法

> tips: 我们目前是停留在AI的应用层，我们做的都是AI的应用开发层上

## ReAct

「推理(Reasoning)+行动(Action)」的循环，也是它名字的由来

大模型先推理，下一步需要干嘛、需要调用什么工具，执行循环调用工具直到整个任务完成

核心执行流程:用户提出需求 →Agent 先思考(推理)要完成这个需求，第一步需要做什么、调用什么工具→执行对应的行动(比如调用 Function call、检索RAG 知识库)》拿到行动的结果(观察)》 再基于结果思考下一步要做什么 》循环往复，直到完成用户的全部需求

## Transformer 架构

举例： 打 打人、打猎、打击、打火（理解文本上下文语意，词语之间关联关系，进而推导生成内容）

Transformer 是目前所有主流大模型的底层基础架构，2017 年由谷歌提出，核心突破是自注意力机制，能让模型理解文本的上下文语义、词语之间的关联关系，从而实现自然语言的理解和生成。
